**Goal:** Reshape the single-file offline HTML report to match Understand-Anything's dashboard look & feel — design-token theming (light/dark), a rich NodeInfo detail panel with method source code, persona-adaptive detail, fuzzy search, and a layer legend — using only vanilla JS, no build step, no external assets.

**Architecture:** `render.py` extends the dashboard JSON payload so each unit node carries its method `source` (sliced from file via the node's `line`/`end_line`), plus a small legend/persona/theme config. `templates/report.html.tmpl` is restyled around CSS-custom-property design tokens with `[data-theme]` and `[data-persona]` on the root, gains a NodeInfo panel populated on node click, a persona button group, a fuzzy-search matcher, a theme toggle, and a layer legend. No changes to the engine (`index.py`/`trace.py`/`tables.py`/`graph.py`/`parsing.py`).

**Tech Stack:** Python 3.10+, pytest, ruff. Report is HTML + CSS + vanilla JS (no framework, no build).

```python
def _method_source(node: dict, max_chars: int = 2000) -> str:
    file = node.get("file")
    start = node.get("line")
    end = node.get("end_line")
    if not (file and start and end):
        return ""
    try:
        lines = Path(file).read_text(encoding="utf-8", errors="replace").splitlines()
    except OSError:
        return ""
    body = "\n".join(lines[start - 1:end])
    if len(body) > max_chars:
        return body[:max_chars] + "…"
    return body
```

In `_dashboard_graph_json`, add `"source": _method_source(node)` to each node dict in the payload comprehension (harmless for table nodes — returns `""`). Keep the existing `<`/`>`/`&` JSON escaping.

```python
def _legend_html(graph: dict) -> str:
    counts = _layer_counts(graph)

    items = "".join(
        f'<span class="legend-item {_layer_class(layer)}">'
```
